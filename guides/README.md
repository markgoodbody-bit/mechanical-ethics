# Guides

These guides expand the Correction Window Test into common use cases.

They are practical guidance, not validation claims.

## Guides

- [`ai_governance_use.md`](ai_governance_use.md) — applying the test to AI systems, AI deployment, and AI governance.
- [`design_patterns.md`](design_patterns.md) — design patterns for making safeguards more live and less theatrical.

## How to use these guides

Use the main test first.

Then use a guide if you are asking a specific question:

- For AI review: use the AI governance guide.
- For system design: use the design patterns guide.
- For public claims and method limits: use the method and falsifier files.

## Boundary

These guides do not certify a system.

They help ask whether correction can still matter before harm hardens.
