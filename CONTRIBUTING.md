# Contributing

This repository is early-stage public work.

Contributions are welcome if they improve clarity, evidence, caveats, examples, or reproducibility.

## Useful contributions

Good contributions include:

- applying the worksheet to a real system;
- correcting unclear wording;
- adding a worked example;
- identifying a missing caveat;
- checking public-record data;
- extending a public-record series;
- pointing out where the test overreads;
- proposing a falsifier;
- identifying an existing framework that already does the same job better.

## Unhelpful contributions

Please do not treat this repository as:

- proof of a complete ethics theory;
- a general anti-AI manifesto;
- a claim that all safeguards are fake;
- a claim that faster review is always better;
- a debate about whether AI is conscious;
- a place for private project mythology.

## Tone

Prefer:

- concrete cases;
- source links;
- short criticism;
- clear uncertainty;
- worked examples.

Avoid:

- sweeping validation;
- sweeping dismissal without a case;
- jargon-heavy rewrites;
- claims that cannot be checked.

## How to challenge the work

The strongest challenge is not "I disagree."

The strongest challenge is:

1. pick one real system;
2. apply the worksheet;
3. show where it fails, adds nothing, or misses the real issue;
4. suggest a narrower or better diagnostic.

## Current priority

The current priority is making the Correction Window Test usable by strangers without private context.

If a reader cannot understand and apply the tool from the public files alone, that is a defect.
