# Mechanical Ethics

## A Human Manual for Acting Under Uncertainty

Status: book draft v0.1. Incomplete. Not canon. Not validation.

## 0.1 What this book is

Mechanical Ethics is a human-facing book about acting after honest seeing.

It begins from a minimal premise: something is here, it perceives partially, it can change something, it does not know everything, and time passes. From there it follows the pressure outward: other things also continue; actions interfere; time closes paths; correction can arrive too late; opportunity can expire; residue remains.

The book does not begin by declaring a moral theory. It tries to earn its first constraints from the structure of action under uncertainty.

```trace
book_promise :=
  start_from_minimal_origin
  + build_to_prescription
  + show_route
  + preserve_limits
```

## 0.2 What this book is not

This book is not a moral oracle. It is not a claim to final moral law. It is not a policy manual. It is not a machine that makes responsibility disappear.

It does not validate TRACE. It does not claim that AI agreement proves anything. It does not turn a description of harm into permission to act.

```trace
book_refusals :=
  no_final_moral_law
  + no_TRACE_validation_claim
  + no_router_permission_slip
  + no_clean_solution_to_dirty_conflict
  + no_fake_precision
```

## 0.3 Reader contract

The reader should be able to use this book without reading TRACE. TRACE is in the walls, not the doorway.

Where the book uses shared terms such as future-space, loss clocks, opportunity clocks, residue, or estimator status, those terms remain TRACE-owned. Mechanical Ethics may explain them for a human reader. It may not redefine them.

The human book keeps the principle in plain form:

```trace
description_is_not_permission :=
  seeing_a_transition
  != authorizing_that_transition
```

The operator interface belongs in case records, use protocols, and appendices. The human argument should not require the reader to understand repository plumbing.

## 0.5 Draft discipline

This draft should grow in wide passes, not polish loops.

First pass: establish the argument motion.
Second pass: add worked examples and stronger transitions.
Third pass: tighten prose.
Fourth pass: pressure-test, cut overclaim, and repair drift.

No part of this book should pretend to settle what it has only framed.
