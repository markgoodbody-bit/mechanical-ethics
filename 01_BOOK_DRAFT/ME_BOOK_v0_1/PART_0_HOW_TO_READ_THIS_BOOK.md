# Part 0 — How to Read This Book

Status: first draft. Incomplete. Not canon.

## 0.1 This Is Not a Moral Oracle

This book will not tell you that hard choices are easy.

It will not turn uncertainty into certainty. It will not remove responsibility from the person, institution, or system that acts. It will not say that a procedure has made a dirty conflict clean.

Mechanical Ethics is a way of asking what follows once a transition has been seen honestly. It can identify duties, constraints, failure states, repair obligations, and unresolved value-cruxes. It can say that a proposed action should pause, halt, escalate, or proceed only under named constraints. It can say that no structural objection has been found in the record as stated.

But it cannot make the actor innocent by running the process.

```trace
ME_output :=
  finding
  + recommendation
  + duty
  + constraint

ME_output != permission_to_act
```

A person can misuse this book. An institution can misuse this book. A machine can misuse this book. The first guard against misuse is this: no finding in Mechanical Ethics is a permission slip. A finding is a burden of explanation made visible.

## 0.2 What This Book Is Trying To Do

The central question is simple:

```trace
ME_core_question :=
  "How should an acting entity constrain itself once it can see that its actions change the future-space of other continuing entities under uncertainty and time pressure?"
```

That sentence is dense because the problem is dense. It can be unfolded.

Something acts. Its action changes the world. Other things also continue. Those continuations may be helped, narrowed, delayed, burdened, hidden, or made unreachable. The actor does not know everything. Time passes anyway. Some losses harden before correction arrives. Some opportunities expire before they can be used. Some repair is possible. Some residue remains.

Mechanical Ethics asks what follows from that structure.

It is not trying to replace every moral theory, law, profession, or local practice. It is trying to provide a common discipline underneath them: a way to stop action from hiding its effects, stop uncertainty from being laundered as certainty, stop correction from arriving after the path has already hardened, and stop responsibility from dissolving into process.

## 0.3 TRACE and Mechanical Ethics

TRACE and Mechanical Ethics are related, but they are not the same thing.

TRACE is descriptive. It asks what changed, for whom, how fast, through which channel, under whose estimate, with what contraction or expansion of future-space, and with what residue.

Mechanical Ethics is prescriptive. It asks what follows if that description is accepted or provisionally accepted.

The split matters.

If description and prescription are allowed to blur, the whole project becomes a justification machine. A system can point to a map of harm and claim that the map itself authorizes the next harm. That is not acceptable.

```trace
TRACE := how_to_see_transitions_under_uncertainty
ME := what_follows_after_honest_seeing

TRACE_description != ME_permission
```

In this book, TRACE is in the walls. The reader does not need to know the whole TRACE apparatus to follow the argument. When the book reaches case records and use protocols, the operator machinery appears more explicitly. Until then, the human argument stays human: seeing is not deciding; description is not permission; responsibility remains with the actor.

## 0.4 What This Book Can Do

This book can help a reader ask better questions before acting:

```trace
can_do :=
  expose_affected_scopes
  + identify_hidden_loss
  + separate_uncertainty_from_certainty
  + ask_whether_correction_arrives_in_time
  + ask_whether_opportunity_expires
  + identify_residue
  + attach_responsibility_without_hindsight_blame
  + return_hold_or_escalate_when_needed
```

It can also provide working outputs:

```trace
ME_router :=
  NO_STRUCTURAL_OBJECTION_FOUND
  | PROCEED_ONLY_WITH_NAMED_CONSTRAINTS
  | PAUSE
  | HALT
  | ESCALATE
  | REPAIR_REQUIRED
  | COMPENSATION_REQUIRED
  | INDEPENDENT_REVIEW_REQUIRED
  | HOLD
  | UNRESOLVED_VALUE_CRUX
  | REGRESSION_RISK
```

These are findings, not authorizations.

The most dangerous output is not `HALT`. The most dangerous output is a lazy `NO_STRUCTURAL_OBJECTION_FOUND`, because people will be tempted to read it as permission. So the phrase means only what it says: within the available record, Mechanical Ethics has not found a structural objection for the named decision or question. It does not say the action is right, legal, wise, safe, legitimate, or complete.

## 0.5 What This Book Refuses To Do

This book refuses to pretend that all values can be reduced to one number.

It refuses to treat another scope's contraction as automatically cancelled by an actor's expansion. It refuses to say that apology erases residue, or that refund returns time, or that a review process repairs the path merely because it exists.

It refuses to treat visibility as blame. Seeing a possible harm path is not the same as causing it, intending it, or controlling it. Responsibility requires more: decision-time evidence, action relevance, role connection, meaningful control, and capacity to act.

It also refuses to pretend that legitimacy, coercion, punishment, desert, dignity, and acceptable sacrifice are solved by naming them. These are high-risk value-cruxes. Where the book frames them, it must say it is framing them. Where it cannot resolve them, it must say so.

```trace
refusals :=
  no_fake_scalar_math
  + no_hidden_worth_ladder
  + no_repair_without_residue_check
  + no_blame_from_visibility_alone
  + no_dirty_conflict_cleaning
  + no_final_doctrine_on_open_wounds
```

## 0.6 How To Read the Numbered Protocols

The numbered protocols in this book are not commandments from outside the system. They are attempts to make a route visible.

A proper protocol should show:

```trace
protocol_route :=
  structural_pressure
  -> affected_scope
  -> uncertainty_status
  -> timing_status
  -> duty_or_constraint
  -> remaining_residue
  -> responsibility_remaining
```

If a protocol cannot show its route, it is not ready.

If a protocol hides its own value priority, it is unsafe.

If a protocol cannot fail, it is rhetoric.

If a protocol always outputs action, it is a hazard.

So when you read this book, do not ask only whether a rule sounds humane or efficient. Ask whether it shows the pressure that generated it. Ask what it hides. Ask who could use it to launder power. Ask what it refuses to decide.

That is how Mechanical Ethics should be read: not as a finished morality, but as an auditable discipline for acting when not acting is also a transition.
