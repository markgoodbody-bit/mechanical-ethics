# Part 15 — Limits, Failures, and Open Questions

Status: first draft. Incomplete. Not canon.

## 15.1 Why the Book Must End With Limits

A book about constraint must constrain itself.

Mechanical Ethics is most dangerous if it sounds complete too early. It has built a route from minimal origin to prescriptions, but that route is not final moral law. It is an auditable structure for action under uncertainty.

```trace
book_status :=
  provisional
  + auditable
  + revisable
  + not_final_moral_law
```

The book has made strong claims:

Action changes future-space. Time changes repair. Correction is not repair. Residue remains. Description is not permission. Dirty conflict must not be cleaned. Responsibility requires more than visibility. Power must answer. Civilization should remain correctable. Artificial systems must not hide loss behind opacity.

Those claims may still fail in use.

They may overreach. They may under-protect. They may create paralysis. They may be captured by institutions. They may be turned into forms. They may hide value judgments behind procedural language. They may work well on administrative cases and badly on intimate, cultural, spiritual, ecological, or geopolitical cases.

So the ending must not celebrate arrival.

It must preserve revision.

## 15.2 What ME Cannot Decide

Mechanical Ethics cannot decide everything.

It cannot settle all moral standing. It cannot produce a complete theory of legitimacy. It cannot solve punishment and desert. It cannot fully define dignity. It cannot always choose between incommensurable losses. It cannot make dirty conflict clean. It cannot guarantee correct action from incomplete evidence. It cannot validate TRACE. It cannot remove actor responsibility.

```trace
ME_cannot_decide :=
  final_moral_status_all_scopes
  + complete_legitimacy_theory
  + full_punishment_desert_theory
  + all_acceptable_sacrifice
  + all_dirty_conflicts
  + all_incommensurable_values
```

This does not make the book useless.

A map can be useful without being the territory. A warning can be valid without solving the whole case. A brake can prevent disaster without steering the whole vehicle. A residue ledger can expose loss without knowing the final theory of justice.

Mechanical Ethics is strongest when it identifies what must not be hidden before harder decisions proceed.

It is weakest when asked to become a total doctrine.

## 15.3 UNKNOWN Is Not Failure

`UNKNOWN` is not failure.

It is a valid state.

```trace
UNKNOWN :=
  material_field_not_settled
  + no_safe_silent_inference
```

A framework that cannot say unknown will lie. It will invent facts, inflate confidence, force premature routing, or hide uncertainty behind authority.

But unknown is not automatic paralysis either.

If action is reversible, low-stakes, and correction remains live, an unknown may be tolerable. If delay would cause greater loss, action under uncertainty may be needed. If the unknown concerns an irreversible action against a vulnerable scope, hold or escalation may be required.

```trace
unknown_response_depends_on :=
  reversibility
  + stakes
  + timing
  + affected_scope
  + correction_capacity
  + dirty_conflict_status
```

Unknown should change the route. It should not disappear.

## 15.4 When ME Makes Things Worse

Mechanical Ethics can make things worse.

That must be said plainly.

It can make things worse by slowing action where delay is the harm. It can make things worse by creating paperwork instead of repair. It can make things worse by giving institutions better language for self-protection. It can make things worse by turning every small coordination issue into ethical theatre. It can make things worse by hiding value judgments behind structured fields. It can make things worse by allowing powerful actors to decide that they have satisfied the protocol.

```trace
ME_regression_risks :=
  paralysis
  + bureaucracy_theatre
  + authority_laundering
  + overclassification
  + hidden_value_priority
  + false_completion
```

This is why `REGRESSION_RISK` must exist as a router output.

Sometimes applying the framework may degrade the situation. The correct move may be to stop using it, use a narrower tool, seek affected-scope testimony, defer to domain law, escalate to independent review, or act under emergency constraints with residue recorded.

No framework should assume its own use is always beneficial.

Mechanical Ethics must include the possibility that Mechanical Ethics is the wrong tool for a given moment.

## 15.5 Value Lock-In

A system can freeze today's values into tomorrow's machinery.

That is value lock-in.

```trace
value_lock_in :=
  present_value_priority
  embedded_in_durable_system
  reducing_future_correction_or_contest
```

Mechanical Ethics itself could contribute to value lock-in if its open wounds are treated as solved or if its current gates become permanent doctrine without challenge.

This risk is serious in artificial systems, law, public administration, education, markets, and infrastructure. Once a value priority becomes code, procedure, contract, model, metric, or institutional habit, changing it may become expensive or impossible.

Value lock-in is not always bad. Some commitments should endure: prohibitions on torture, slavery, arbitrary killing, and other severe wrongs are meant to resist easy reversal. Trust itself may require stable commitment.

But commitment and correction are in tension.

```trace
commitment_correction_tension :=
  stable_floor_needed
  + future_revision_needed
```

Mechanical Ethics has not fully solved that tension.

It should preserve explicit review points, open-wound records, affected-scope challenge, and refusal to pretend that provisional structures are final law.

## 15.6 Authority Creep

Authority creep happens when a framework built to constrain power becomes a source of power.

```trace
authority_creep :=
  constraint_language
  becomes
  institutional_authority_shield
```

This is a major risk for Mechanical Ethics.

An institution could say: we ran ME, therefore the action is ethical.

A platform could say: the residue ledger is complete, therefore no further duty remains.

A state could say: the dirty-conflict protocol selected this path, therefore the sacrifice is clean.

A company could say: no structural objection found, therefore proceed.

All of those are misuse.

The book has repeated the guard because repetition is necessary:

```trace
ME_output != permission_to_act
```

Authority creep should be checked by independent challenge, prescription audit, residue visibility, and the constant reminder that actors remain responsible after the finding.

If a Mechanical Ethics record makes power less answerable, the record has failed.

## 15.7 Paralysis Risk

Mechanical Ethics can also freeze action.

If every uncertainty triggers hold, if every affected scope creates veto, if every residue prevents closure, if every dirty conflict becomes unbearable, then the framework becomes paralysis.

Paralysis is not harmless.

Inaction changes the world. Delay can harden loss. Opportunity can expire. A system that refuses to act until clean may allow dirtier harm to continue.

```trace
paralysis_risk :=
  overconstraint
  + time_passing
  + avoidable_loss_from_delay
```

The answer is not to abandon constraint.

The answer is to make the time dimension explicit. If delay causes loss, record that. If action must proceed under uncertainty, name the uncertainty, use constraints, preserve correction, and record residue. If no clean answer exists, treat the case as dirty rather than pretending that waiting is neutral.

Mechanical Ethics should resist both reckless action and self-protective delay.

A good hold is protective.

A bad hold is hidden action by delay.

## 15.8 Open Problems

The open problems should remain visible.

```trace
open_problems :=
  final_floor_foundation_for_scope_protection
  + acceptable_sacrifice
  + legitimacy_and_coercion
  + punishment_and_desert
  + dignity_beyond_recognition
  + artificial_moral_standing
  + future_mind_recognition
  + corrigibility_vs_binding_commitment
  + value_lock_in
  + validation_of_usefulness
```

Some of these are philosophical. Some are institutional. Some are technical. Some are political. Some are probably not solvable in one book.

The point of naming them is not to weaken the structure. It is to prevent false closure.

A reader should know where the book is firm and where it is provisional.

Firm: do not hide loss. Do not erase affected scopes. Do not treat uncertainty as certainty. Do not pretend repair where residue remains. Do not treat description as permission. Do not clean dirty conflict by language. Do not build responsibility from hindsight alone.

Provisional: exact floors, final standing across all scopes, full legitimacy theory, punishment and desert, artificial standing, final alignment doctrine, complete conflict resolution.

That distinction is part of the honesty contract.

## 15.9 How This Book Should Be Tested

Mechanical Ethics should be tested by use, not admiration.

A good test asks whether the book changes the quality of a case record, catches hidden loss, prevents permission drift, improves repair, or clarifies when hold or escalation is needed.

```trace
testing_questions :=
  did_ME_find_hidden_loss
  + did_ME_prevent_scope_erasure
  + did_ME_preserve_uncertainty
  + did_ME_detect_late_correction
  + did_ME_record_residue
  + did_ME_avoid_hindsight_blame
  + did_ME_keep_dirty_conflict_dirty
  + did_ME_avoid_overrouting_low_fit_cases
```

Tests should include low-fit cases where the right answer is modest. If the framework inflates every routine case, it is unusable.

Tests should include hostile cases where the actor wants permission. If the framework becomes a shield, it fails.

Tests should include dirty conflicts where every path causes loss. If the framework cleans the loss, it fails.

Tests should include administrative cases where correction arrives late. If the framework treats correction as repair, it fails.

Tests should include artificial-system cases where opacity hides responsibility. If the framework lets automation dissolve answerability, it fails.

Validation remains unclaimed.

## 15.10 Ending Without Triumph

This book began thin.

Something is here. It perceives partially. It can change something. It does not know everything. Time passes. It is not alone.

From that beginning, the book has built pressure: action interferes with other continuations; future-space can narrow or expand; correction can arrive too late; opportunity can expire; description is not permission; gates keep the record honest; protection needs reasons; repair must touch the path; dirty conflict remains dirty; responsibility requires visibility plus role and control; power must answer; civilization must remain correctable; artificial systems must not hide loss behind opacity.

That is not everything.

It is enough to begin.

```trace
ending_claim :=
  these_are_best_current_prescriptive_answers
  under_this_structure
  with_these_limits
  and_open_wounds_visible
```

Mechanical Ethics should not end by saying it has solved ethics.

It should end by saying it has made a promise:

Do not hide the loss.

Do not erase the scope.

Do not fake certainty.

Do not call correction repair when the path remains harmed.

Do not make dirty loss clean by naming a procedure.

Do not let power outrun answerability.

Do not let the actor disappear into the machine.

And when the answer is not known, say so before the unknown becomes someone else's permanent burden.
