# Part 8 — Correction, Repair, and Residue

Status: first draft. Incomplete. Not canon.

## 8.1 Correction Is Not Repair

Correction and repair are not the same thing.

Correction changes an official state. Repair changes the harmed path as far as possible.

```trace
correction :=
  record_or_decision_changed

repair :=
  harmed_path_touched
  + future_space_restored_where_possible
  + recurrence_risk_reduced
  + residue_recorded
```

A file can be corrected while a person's future remains narrowed. A payment can be restored after debt, fear, eviction risk, or family damage has already occurred. A decision can be reversed after a job, school place, treatment window, or appeal route has already closed.

Correction matters. It is better than refusing error. It may be the first step toward repair.

But it should not be allowed to impersonate repair.

The distinction matters because institutions often treat correction as closure. The system finds the mistake, changes the record, issues a note, possibly pays compensation, and moves on. The affected person is expected to accept that the official state now matches what should have happened earlier.

But earlier is gone.

Mechanical Ethics asks what the correction failed to return.

```trace
repair_question :=
  what_path_was_harmed
  + what_can_still_be_restored
  + what_cannot_be_restored
  + what_must_change_to_prevent_recurrence
```

This is the beginning of repair discipline.

## 8.2 Repair Must Touch the Path

Repair must touch the path that caused the harm.

It is not enough to change language around the path. It is not enough to apologise for the path. It is not enough to compensate for one case while preserving the same machinery for the next case.

```trace
repair_must_touch_path :=
  mechanism_changed
  or burden_routed_back
  or reachability_restored
  or evidence_preserved
  or recurrence_risk_reduced
```

If the harm came from delay, repair must address delay.

If the harm came from unreachable appeal, repair must address reachability.

If the harm came from hidden evidence, repair must address evidence and disclosure.

If the harm came from contaminated estimation, repair must address estimator independence and challenge.

If the harm came from local reward pull, repair must address what the system is rewarded for.

A system that harms through timing cannot repair only through apology.

A system that harms through burden cannot repair only through information.

A system that harms through opacity cannot repair only through speed.

A system that harms through recurrence cannot repair only the last visible case.

Mechanical Ethics therefore treats repair as path-specific.

The question is always: what mechanism narrowed future-space, and has that mechanism been constrained?

## 8.3 Slow the Write

Some systems write faster than people can answer.

They write records. They write classifications. They write debts. They write refusals. They write risk scores. They write exclusions. They write defaults. They write police attention. They write medical priority. They write school futures. They write immigration status. They write social trust.

When writing is fast and correction is slow, harm can harden before the affected scope can contest.

```trace
write_speed_problem :=
  T_write
  < T_detect + T_route + T_correct
```

The first repair may be to slow the write.

This does not mean every system must stop. It means a system should not be allowed to make irreversible or hardening changes at a speed that outruns meaningful correction.

Slowing the write can mean temporary pause, staged effect, reversible status, warning period, human review before hardening, automatic buffer, independent check, or delayed enforcement.

```trace
slow_write_options :=
  pause
  + stage
  + buffer
  + reversible_status
  + pre_hardening_review
  + independent_check
```

The point is to keep correction real while there is still something to correct.

If the write cannot be slowed because urgent action is required, then the record should say why, what loss may be created, what residue may remain, and what repair duty follows.

Speed is not neutral. Speed allocates power.

## 8.4 Route the Burden Back

Harm often travels by burden shift.

A system creates uncertainty, but the affected person must prove certainty. A model makes an error, but the worker must disprove it. A record is wrong, but the claimant must gather evidence. A public body delays, but the citizen carries rent, hunger, fear, or lost opportunity. A platform acts automatically, but the user must navigate opaque appeal.

```trace
burden_shift :=
  system_uncertainty
  routed_to
  affected_scope
```

Not every burden shift is wrong. Shared systems require participants to provide information, cooperate, and sometimes wait. But burden becomes ethically serious when it is forced onto those least able to carry it, especially where the system caused or controls the uncertainty.

Repair should route burden back where possible.

```trace
burden_return :=
  system_rechecks
  + system_preserves_evidence
  + system_contacts_sources
  + system_pauses_enforcement
  + system_funds_access
  + system_explains_decision
```

The affected person should not have to become a lawyer, engineer, data scientist, doctor, or bureaucratic expert merely to stop a system from injuring them through its own uncertainty.

This does not remove all responsibility from affected persons. It says the route must be proportionate to power, access, and cause.

If the system created the fog, it should not make the exposed person carry the whole lantern.

## 8.5 Preserve Contradiction

Repair requires contradiction to survive.

A system that destroys contradiction destroys repair. If the affected person's account is overwritten, if records vanish, if informal warnings are ignored, if dissent is treated as noise, if evidence is discarded after closure, then later correction becomes theatre.

```trace
preserve_contradiction :=
  keep_dispute_visible
  + preserve_evidence
  + prevent_official_story_closure
  + allow_challenge_before_hardening
```

Contradiction is not always truth. People can be wrong. Witnesses can misremember. Affected scopes can misunderstand the mechanism. Records can be messy.

But contradiction is often where the path first becomes visible.

A complaint, a refusal, a mismatch, a repeated anomaly, a local witness, a person saying "this cannot be right" — these may be the first signals that the official model has lost contact with reality.

Mechanical Ethics does not say contradiction should always win.

It says contradiction should not be erased before it can be tested, especially when the official path can harden faster than the challenge route.

A system that cannot preserve contradiction cannot honestly claim robust correction.

## 8.6 Keep Contestability Live

Contestability is the practical ability to challenge a transition before it hardens.

It is not enough that a right to contest exists somewhere in the rules.

```trace
contestability :=
  known_route
  + reachable_route
  + usable_information
  + time_before_hardening
  + authority_to_change_path
```

A challenge route is weak if the affected scope does not know it exists, cannot understand it, cannot afford it, cannot access it, cannot gather evidence for it, or cannot get a decision before loss hardens.

Mechanical Ethics treats contestability as a living property, not a formal checkbox.

If contestability is live, a system may be able to act with less risk. If contestability is dead, action carries more ethical burden.

Keeping contestability live may require notice, translation, advice, time, evidence disclosure, pause, independent review, or temporary protection against enforcement.

```trace
contestability_support :=
  notice
  + explanation
  + evidence_access
  + time
  + affordable_route
  + pause_or_buffer
  + decision_authority
```

The point is not to make every action litigable forever. The point is to prevent systems from saying "you could have challenged" where challenge was practically unreachable until after the damage was done.

## 8.7 Proof Is Not Theatre

Institutions like proof of reform.

They produce new policies, training, dashboards, forms, audits, statements, committees, and lessons learned. Some of these matter. Some are theatre.

Proof is theatre when it demonstrates that something was said, not that the path changed.

```trace
proof_theatre :=
  visible_reform_signal
  + unchanged_harm_mechanism
```

Mechanical Ethics asks for path proof.

If delay caused harm, show delay changed.

If hidden evidence caused harm, show evidence access changed.

If burden shift caused harm, show burden routing changed.

If local reward caused harm, show reward changed.

If no one could answer, show responsibility attachment changed.

If correction was unreachable, show reachability changed.

```trace
path_proof :=
  mechanism_before
  + mechanism_after
  + affected_scope_test
  + recurrence_risk_reduction
```

This does not require impossible certainty. It requires the reform to touch the mechanism it claims to repair.

A sincere institution can still produce theatre if it measures the wrong thing.

A cynical institution can produce theatre deliberately.

The test is the same: did the path change?

## 8.8 Residue Remains

Some things are not restored.

Time is not always returned. Trust is not always restored. Opportunity is not always recreated. Health is not always recovered. A record scar may remain. A relationship may remain damaged. A person may remain altered by carrying the burden. A community may remember. An animal killed in a necessary cull does not become alive because the policy was justified. A future-space contraction may leave a shape that compensation cannot erase.

Mechanical Ethics calls this residue.

```trace
residue :=
  loss_not_restored
  + time_not_returned
  + trust_damage
  + opportunity_expired
  + record_scar
  + recurrence_risk
  + acknowledgement_needed
```

Residue is not a reason to give up on repair. It is a reason to stop lying about repair.

The record should say what remains.

A residue ledger does not have to be long. It has to be honest.

```trace
residue_ledger :=
  what_remains
  + who_carries_it
  + whether_it_can_be_reduced
  + who_still_owes_attention
```

Without residue, institutions overclaim. They turn partial repair into full closure. They count money as time. They count apology as restoration. They count compliance as trust.

Residue keeps the remainder visible.

## 8.9 Duties After Late Correction

When correction arrives late, duties do not disappear.

They change shape.

```trace
late_correction_duties :=
  acknowledge_lateness
  + identify_expired_opportunity
  + repair_path_where_possible
  + compensate_where_appropriate
  + record_residue
  + reduce_recurrence
```

If a person wins an appeal after the loss has hardened, the system should not say only "appeal allowed." It should ask what the late route cost.

If a record is corrected after a decision was already made, the system should ask what opportunity the wrong record affected.

If payment is restored after debt accumulated, the system should ask what debt, stress, penalty, trust loss, or practical narrowing remains.

If a policy is changed after many cases, the system should ask who carries unrepaired residue from the earlier path.

Late correction may still matter. It can restore some options, acknowledge truth, reduce future recurrence, and create evidence for responsibility.

But lateness must be part of the duty record.

A system that corrects late and records no lateness is still hiding part of the harm.

## 8.10 Stewardship Over Time

Repair is not a moment.

Some duties persist.

A changed policy can decay. A corrected record can be copied from an old database. A compensated person can face recurrence. A review channel can become slow again. A model can drift. A public commitment can fade after attention moves on.

```trace
stewardship :=
  repair_over_time
  + recurrence_monitoring
  + memory_preservation
  + renewed_contestability
```

Stewardship is the duty to keep repair from becoming a performance that expires as soon as pressure drops.

This matters especially in large systems. Institutions forget. Staff change. Vendors change. Models update. Laws shift. Budgets tighten. Emergency patches become ordinary routines. The old path may return in a new form.

Mechanical Ethics therefore treats repair as requiring memory.

Not endless punishment. Not permanent suspicion. Memory.

A system that harmed through timing should remember timing. A system that harmed through opacity should remember opacity. A system that harmed through burden shift should remember burden shift. A system that harmed through contaminated estimation should remember who was allowed to estimate and why that failed.

```trace
repair_memory :=
  harm_mechanism
  + repair_action
  + residue
  + recurrence_signal
  + owner_of_follow_up
```

Part 8 ends with a practical claim.

Correction without repair is incomplete.

Repair without residue is dishonest.

Residue without stewardship becomes abandonment.

The next part turns to the hardest cases: where every available action causes loss, and the task is not to find a clean answer but to stop the dirty answer from pretending to be clean.
