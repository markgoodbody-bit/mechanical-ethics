# Value Spine Integration v0.1

Status: structural integration surface for the ME Book v0.1 first wide-pass draft. Not canon. Not validation. Not a final moral theory.

Purpose: run the good/bad/positive-becoming layer through the book without smuggling certainty, personhood, law, rights, or a finished theory of the Good into the foundation.

This file is not a case study request. Case pressure remains useful later. The present task is structural: build the evaluative grammar that lets the book explain positive value as well as harm, repair, residue, and anti-laundering.

## 1. Governing constraints

```trace
governing_constraints :=
  honesty_first
  + uncertainty_as_primitive
  + middle_out_only
  + no_contemporary_story_references_in_book
  - final_moral_theory_smuggling
  - fake_certainty
  - harm_only_frame
```

ME must not claim final access to the Good. It may make scoped, timed, uncertain, revisable, residue-aware claims about transitions.

```trace
ME_can_say :=
  this_transition_opens_or_closes_these_futures
  for_these_scopes
  under_this_uncertainty
  with_these_remaining_costs
  therefore_these_constraints_or_duties_follow

ME_cannot_say :=
  final_good_declared
  + final_bad_declared
  + uncertainty_erased
  + all_costs_cancelled
  + affected_scopes_settled_forever
```

## 2. Middle-out start

Start lower than moral theory:

```trace
middle_out_start :=
  something_exists
  + has_state
  + state_can_change
  + change_can_open_or_close_future
  + perception_is_partial
  + uncertainty_never_zero
```

Before morality appears, a state can still become more or less viable. An entity can be more oriented or less oriented, more capable or less capable, more able to repair or less able to repair. This is not yet law, rights, duty, personhood, or virtue. It is the structural ground from which ethical pressure later grows.

## 3. Good and bad as candidate transition directions

Avoid overclean definitions:

```trace
do_not_use :=
  good := future_space_expansion
  bad := future_space_contraction
```

That is too certain and too crude. Raw future-space expansion is not automatically good. More options may include addictive, predatory, destructive, coercive, or self-closing options.

Use candidate language:

```trace
good_candidate :=
  transition
  that_appears_to_support_viable_becoming
  for_named_scope_or_scopes
  under_stated_uncertainty
  with_costs_and_residue_visible

bad_candidate :=
  transition
  that_appears_to_damage_or_close_viable_becoming
  for_named_scope_or_scopes
  under_stated_uncertainty
  especially_when_hidden_unrepairable_or_forced
```

This keeps uncertainty inside the value claim.

## 4. Viable becoming

```trace
viable_becoming :=
  continuation
  + orientation
  + capacity
  + relation
  + repairability
  + meaningful_future_space
  under_uncertainty
```

Viable becoming is not perfection, happiness, purity, productivity, or unrestricted choice. It is the live structure by which an entity can continue, orient, learn, relate, repair, choose meaningfully, and remain open to further becoming.

The book should treat viable becoming as a structural bridge, not as a completed theory of flourishing.

```trace
ME_value_claim_limit :=
  ME_does_not_prove_final_good
  but_identifies_conditions
  without_which_good_cannot_be_pursued
```

## 5. Good/bad record discipline

Every good/bad claim must ask:

```trace
good_bad_record :=
  good_for_whom
  + bad_for_whom
  + over_what_time
  + at_what_scope
  + with_what_cost
  + with_what_repair_path
  + with_what_confidence
```

This prevents clean labels from hiding mixed transitions.

```trace
mixed_transition :=
  good_for_A
  + bad_for_B
  + uncertain_for_C
  + residue_for_D
```

```trace
ethical_pressure_begins :=
```

```trace
ethical_good_candidate :=
```

### 7.1 Care

```trace
care :=
  another_scope_future_space
  represented_inside_actor_future_map
  with_protective_or_expansive_weight
```

Care is not only non-harm. Care includes another continuation in one's own map of possible futures.

### 7.2 Trust

```trace
trust_positive :=
  reliable_return_path
  + non_punitive_error_surface
  + predictable_care_under_stress
  + expanded_willingness_to_act
```

Trust is future-space infrastructure. It makes risk, learning, asking, repair, cooperation, and return more reachable.

### 7.3 Play

```trace
play :=
  voluntary_uncertainty
  + bounded_risk
  + role_experiment
  + feedback
  + repairable_failure
  + future_capacity_growth
```

Play is not decorative. It is one primitive form of correctable action under safe uncertainty.

### 7.4 Cooperation

```trace
cooperation :=

  + role_coordination
  + preserved_exit_or_contest
  + repairable_misalignment
```

Cooperation is positive future-space expansion across scopes without requiring erasure of separate interests.

### 7.5 Love candidate

```trace
love_candidate :=
```

## 8. False goods

ME must track false good as carefully as hidden harm.

```trace
false_good_patterns :=
  pleasure_that_addicts
  + efficiency_that_erases
  + safety_that_dominates
  + certainty_that_blinds
  + repair_that_only_changes_record
  + benefit_that_hides_burden
  + freedom_that_removes_correction
```

Critical distinctions:

```trace
local_reward != structural_good
local_pain != structural_bad
raw_options != viable_future_space
constraint != oppression_by_default
pleasure != flourishing_by_default
benefit != absolution
description != permission
```

## 9. Good constraint

Constraint is not automatically bad. Some constraints preserve wider viable becoming.

```trace
good_constraint :=
  local_option_narrowing
  that_preserves_or_expands
  longer_horizon_viable_future_space
  with_visible_burden
  + correction_route

bad_constraint :=
  option_narrowing
  through_domination
  + opacity
  + false_necessity
  + erased_scope
  + blocked_correction
```

This prevents the book from collapsing into raw option-counting.

## 10. Benefit is not absolution

Pair this with `description_is_not_permission`.

```trace
benefit_is_not_absolution :=
  claimed_good
  must_still_name
    affected_scopes
    + what_expands
    + what_contracts
    + what_burden_remains
    + what_correction_stays_live
```

A claimed good can be real and still incomplete. It can expand future-space for one scope while narrowing it for another. It can reduce one harm while creating residue elsewhere. Therefore benefit must be audited, not worshipped.

## 12. Value transition model

```trace
value_transition_model :=
  candidate_good :=
    viable_becoming_supported

  candidate_bad :=
    viable_becoming_damaged

  harm :=
    material_contraction_or_corruption_of_viable_becoming

  repair :=
    attempt_to_reopen_or_restore_viable_becoming

  residue :=
    viable_becoming_not_restored
    + trust_not_rebuilt
    + opportunity_expired
    + burden_remaining
```

This connects positive value, harm, repair, and residue without making harm the whole field.

```trace
  + Part_14_case_records_include_positive_audit

```

### Part 1

Introduce good and bad before morality: some state changes support continuation, orientation, capacity, relation, repairability, and meaningful future-space; some damage them. Keep this pre-ethical and uncertain.

### Part 2

Show the hinge: good for one entity can be bad for another. Ethics begins when viable-becoming claims collide under uncertainty.

### Part 3

Make `local_reward != structural_good` load-bearing. Selection systems can learn false goods.

### Part 4

Distinguish raw future-space from viable future-space. More options are not automatically better.

### Part 5

Add the paired firewall: benefit is not permission. Seeing a claimed good does not authorize the transition.

### Part 6

Reframe gates as preventing false goods as well as hidden harms.

### Part 7

Explain protection as preserving the conditions for viable becoming under uncertainty, not as settled personhood doctrine.

### Part 8

Define repair as reopening viable becoming where possible, while recording what remains closed.

### Part 9

Dirty conflict means not all viable-becoming claims can be preserved at once. The conflict remains dirty even when action is necessary.

### Part 10

Responsibility attaches when an actor can see and affect openings or closures of viable becoming under decision-time evidence and meaningful control.

### Part 11

Power is the capacity to shape another scope's viable becoming. Legitimacy remains framing-not-solving, but the danger is now clearer.

### Part 12

Civilization is shared correctability plus shared viable becoming across time.

### Part 13

AI local reward or benchmark success is not structural good. Artificial systems must not optimize false goods while hiding burdens.

### Part 14

Case records must track claimed benefit as well as loss: what expands, what contracts, what remains burdened, what correction stays live.

### Part 15

The final theory of good remains open. ME's honest contribution is not final moral law; it is tracking how transitions open or close viable becoming under uncertainty.
