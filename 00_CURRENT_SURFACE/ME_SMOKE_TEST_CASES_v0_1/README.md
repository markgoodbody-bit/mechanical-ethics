# Mechanical Ethics Smoke Test Cases v0.1

Status: smoke-test surface. Not validation. Not canon. Not book prose.

Purpose: run very small cases through the minimum viable case record before drafting full worked examples or prose chapters.

## Why this exists

The build should not get bogged down in polishing planning surfaces. The next useful pressure is whether the interface schema can turn on a small case without overclaiming, over-routing, or becoming unusable.

```trace
smoke_test_goal :=
  test_schema_turns
  + detect_overreach
  + detect_router_permission_drift
  + keep_case_small
```

## Files

- `ME_SMOKE_TEST_01_ROUTINE_MEETING_MOVE_v0_1_2026_06_27.md` — low-stakes hypothetical negative-control case.

## Status rule

A smoke test is not evidence that ME works generally. It only checks whether the current schema can be filled consistently and whether it returns a modest result when the case is modest.
